## Intel Survey Finds Maintainer Burnout & Documentation Top Open-Source Challenges

Written by [Michael Larabel](https://www.michaellarabel.com/) in [Intel](https://www.phoronix.com/linux/Intel) on 20 February 2024 at 02:35 PM EST. [22 Comments](https://www.phoronix.com/forums/node/1444575)

![](_page_0_Picture_2.jpeg)

At the end of last year Intel hosted [a survey of open-source developers](https://www.phoronix.com/news/Intel-Open-Source-Survey) to collect their feeback on various open-source software issues. Intel's 2023 Open Source Community Survey is all wrapped up, the data tallied up, and the results emailed out today to participants.

I haven't seen Intel post this data on any public web page yet, but per the email, below are all of the results collected from their open-source survey for 2023.

![](_page_0_Figure_5.jpeg)

To hardly any surprise, the survey found having an acceptable open-source license was the most important open-source aspect. The license was followed by maintainer responsiveness, activity volume, welcoming community, and then the established policies and documentation.

|                                           | Top Open Source Challenges Faced                                             |     |
|-------------------------------------------|------------------------------------------------------------------------------|-----|
| Top<br>Open Source<br>Challenges<br>Faced | Maintainer burnout                                                           | 45% |
|                                           | Documentation and onboarding                                                 | 41% |
|                                           | Maintaining sustainability                                                   | 37% |
|                                           | Building community engagement                                                | 32% |
|                                           | Insufficient budgets and program costs                                       | 31% |
|                                           | Technical debt (contributors do not finish and organizers need to evolve it) | 30% |
|                                           | Quality assurance                                                            | 27% |
|                                           | Security and vulnerability management                                        | 23% |
|                                           | Compatibility and interoperability                                           | 22% |
|                                           | Licensing and compliance                                                     | 16% |
|                                           | Legal and intellectual property issues                                       | 14% |
|                                           | Lack of training opportunities                                               | 14% |
|                                           | Other                                                                        | 8%  |
|                                           | None of the above                                                            | 3%  |

Of those participating in the survey, Intel found the top open-source challenge faced was maintainer burnout at 45%... That was followed by documentation/onboarding at 41% and then maintaining sustainability at 37%.

![](_page_1_Picture_2.jpeg)

Another result of little surprise was finding the open-source project of greatest interest being Linux itself. That was followed by Rust, Clang/LLVM, GCC, and then Kubernetes.

![](_page_2_Figure_0.jpeg)

And as a sign of the times, of course AI was brought up in the survey.... And 82% found opensource in AI being either extremely or very important.

![](_page_2_Figure_2.jpeg)

Of those that participated in this Intel open-source survey, 30% of those contributed to open-source projects more than once per week.
